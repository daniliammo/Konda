# CLAUDE.md — рабочие правила и решения проекта Konda

Транспилятор `.конда` → C. Русскоязычный язык с целью: **программист не может
допустить ошибку управления памятью**, при этом в релизе — скорость уровня C.

## ГЛАВНОЕ ПРАВИЛО: фиксировать важное в документации

Всё, что должно пережить `/compact` и не «потеряться» между сессиями, **сразу
записывается в документацию или в этот файл** — не держим важное только в
контексте диалога:

- **Принятое пользователем решение** (в т.ч. ответы на AskUserQuestion), смена
  направления, инвариант дизайна → в раздел «Принятые решения» этого файла.
- **Реализованная возможность / архитектурный факт** → в `README.md`
  (пользовательский взгляд) и/или `docs/АРХИТЕКТУРА_ТРАНСПИЛЯТОРА.md` (внутренний).
- Обновляя код, **сразу актуализируй доки** (в т.ч. §«Что уже/пока не
  поддерживается» и «Ближайший план» в README) — устаревшая дока хуже её
  отсутствия.
- Перед изменением синтаксиса — **сначала расширить тесты** (`tests/run.sh`).

## Инварианты дизайна (не менять без явного согласия пользователя)

1. **Безопасность — статическая, нулевая рантайм-цена.** Любая новая проверка
   безопасности — на этапе компиляции (или тип), чтобы **релиз работал так же
   быстро, как C**. Единственная рантайм-проверка — границы срезов/массивов/argv
   (её сжимаем статическими доказательствами, а не убираем).
2. **Небезопасное — только в `небезопасно { }`** (счётчик `глубина_небезопасно`,
   в C — обычный блок, нулевая цена). За гейтом: сырой указатель
   (индексация/арифметика), реинтерпретация указателя, доступ к члену `союз`а,
   небезопасные функции libc.
3. **`printf` НЕ трогать** — пользователь позже сделает свою stdlib; формат-строка
   передаётся как есть по дизайну.
4. **Глобальных изменяемых переменных НЕТ** — верхний уровень принимает только
   директивы, функции, пространства и типы. Это **несущее** свойство: вместе с
   «нет сырого `&`» и «нет замыканий» оно делает разделяемое изменяемое состояние
   *неназываемым*, откуда и берётся «гонки невозможны по построению» (модель
   потоков опирается на него напрямую). Добавить глобалы = сделать гонки
   выразимыми и молча обрушить безопасность потоков. Константы верхнего уровня —
   отдельная фича, допустимая, только пока они строго иммутабельны.
5. **Максимизировать статические проверки.**

## Основные принципы (выведены из истории коммитов)

- **Фича = полный вертикальный срез в одном коммите.** Каждая возможность идёт
  сразу через `лексер → парсер → семантика → кодоген → IDE-hover` **плюс** тесты в
  `tests/run.sh` **плюс** обновление `README.md` и `docs/АРХИТЕКТУРА_ТРАНСПИЛЯТОРА.md`.
  В истории почти каждый коммит трогает все четыре — не оставляй фичу без тестов и
  доков. Ловушка парсера: токен, которым может ЗАКАНЧИВАТЬСЯ оператор, обязан быть
  в `токен_завершает_оператор`, иначе перевод строки после него съедается.
- **Предпочитать проверки без рантайма** («Дополнительные проверки без рантайма»,
  «больше статических проверок») — статика вместо рантайм-стоимости везде, где
  можно; рантайм-guard'ы (деление/сдвиг/переполнение) — только в отладке.
- **Безопасность памяти — непрерывная работа.** Регулярные фиксы double-free,
  деструкторов, use-after-move, transitive-destruction; каждый — с тестом-регрессией.
- **Урок (17.07): любой анализ, снимающий проверку, обязан видеть ВСЕ способы
  мутации.** `собрать_изменяемые` собирал только присваивания и не видел
  `изменяемый`/`вывод` — переменная считалась «известной константой», проверка
  границ снималась по ложному основанию, ASan ловил heap-buffer-overflow.
  Мутация без присваивания в коде — это `изменяемый`/`вывод` (и только они:
  глобалов и сырого `&` нет). Тесты — `мутация_вывод`/`мутация_изм`.
- **Безопасен и сам транспилятор.** `cc` запускается через `execvp` с готовым argv
  (не `system()` со строкой — закрыт command injection через имя файла); парсер
  защищён от переполнения стека на патологических выражениях.
- **Русские ключевые слова, контекстные где нужно.** `и`/`или` вместо `&&`/`||`;
  `и` — контекстное (остаётся валидным именем переменной-счётчика). C-запись тоже
  принимается. Новое ключевое слово не должно без нужды отнимать имя у программиста.
- **Диагностики — на русском, с позицией** (`строка`/`колонка` из `Токен`), и в
  **батч-режиме**: семантика собирает все ошибки за прогон (парсер пока стоп-на-первой).
- **Нет сырого `&`.** Мутация переменной вызывающего — через `изменяемый`/`вывод`
  (ref/out, кодоген сам ставит `&`/`*`); возврат указателя — только borrow
  (память переживёт возврат). Так исключены висячие указатели без borrow-checker'а.
- **IDE-first.** Есть стабильный API `libkonda_ide.c` (анализ в памяти без файлов и
  без `cc`) для language server — не ломать его сигнатуры при правках семантики.

## Принятые решения (ответы на AskUserQuestion и смены курса)

- **Null-безопасность (пункт 3 плана): «ненулевые указатели по умолчанию».**
  Обычный `T*` не может быть `нуль` (ни `= нуль`, ни без инициализатора);
  nullable — только `возможно<T*>`, разворот под охранником `если (п != нуль)` или
  `если (п == нуль) … иначе`. Для этого добавлен оператор `!=`.
- **Охват `возможно` — только указатели.** Опция-над-значениями
  (`возможно<целое>` с флагом присутствия) намеренно **отложена** — отдельная
  фича, другое представление.
- **Конвертер C→Konda — отдельный проект `konda-from-clang`** (сосед этого репо,
  Python + `clang -ast-dump=json`). В этом репо конвертера **нет**: проверки — в
  транспиляторе, конвертер только переводит и сам себя проверяет прогоном
  транспилятора (verify-by-default). Ручное управление памятью → `malloc`→
  `выделить`/`срез`, `free` снимается (autofree).
- **Интерфейс библиотек (безопасный путь 2):** типизированный `.и` c примитивами,
  `срез`, указателями и владением; транспилятор сам линкует `.so` (rpath).
- **Макросы — только безопасная условная компиляция** (`макросы.c`): `#if/#ifdef/
  #ifndef/#elif/#else/#endif/#define` (+ русские `#если/#опр/#не_опр/#иначе_если/
  #иначе/#конец/#определить`), `-D ИМЯ[=знач]`. Это выбор веток AST по флагам, не
  текстовая подстановка → включённый код проходит все проверки. **Функциональных
  макросов и подстановки значений в код НЕ делать** — небезопасно; для констант — `конст`.
- **Имена в C — как есть, без манглинга/неявных имён.** Пользователь не хочет
  транслитерации/переименования функций: кириллические идентификаторы уходят в C
  как есть (расчёт на UTF-8-идентификаторы C23). Значит роадмап-пункт «стабильная
  транслитерация в ASCII» **снят для функций** — не генерировать неявные имена.
- **Побитовые операции `& | ^ << >> ~` — реализованы** (целочисленные, нулевая
  рантайм-цена, статически безопасны; float/указатель/срез отвергаются). Ложатся
  в инвариант «безопасность статическая, скорость C». Приоритеты как в C.
- **Потоки: модель «передача владения», без разделяемого состояния.** (Ранее
  потоки были отложены; курс изменён.) Поток получает данные **только по
  значению или перемещением** (`владеет`-семантика). Разделяемое изменяемое
  состояние **нельзя выразить** → гонки данных невозможны *по построению*,
  статически, с нулевой рантайм-ценой (инвариант №1). Опирается на готовый
  path-sensitive анализ move в `владение.c`. Синтаксис: `поток<T>`,
  `запустить(ф, арг)`, `присоединить(п)`. Статические правила: функция потока
  НЕ может иметь `изменяемый`/`вывод` (это указатели в кадр вызывающего →
  повисли бы + гонка); аргумент-срез перемещается (у запускающего доступа нет);
  `присоединить` потребляет дескриптор (двойной join — use-after-move).
  **Shared-memory модель в стиле Rust отвергнута**: она требует borrow-checker,
  а он сознательно исключён из дизайна Konda.
- **Каналы `канал<T>` — ОТЛОЖЕНЫ** (осознанно, а не «не успели»). Они дают ровно
  одно: общение во время жизни потока (стриминг, пул воркеров, конвейеры).
  Разделение труда: fork-join `запустить`/`присоединить` покрывает независимые
  задачи, а параллелизм данных и map-reduce — `параллельно для` **с редукцией**.
  Цена каналов высока: (1) появляется **рантайм-библиотека** (мьютекс+condvar+
  буфер) — против духа «генератор обычного C без своего рантайма»; (2) вносят
  **взаимоблокировки** — первый класс багов, который Konda статически
  предотвратить НЕ может (обещание «нельзя допустить ошибку памяти» формально
  цело, но защиты нет). Вернуться к ним, когда появится реальная задача со
  стримингом.
  *(Исправлено 17.07: раньше здесь стояло, что fork-join покрывает и map-reduce.
  Это было неверно — reduce был запрещён как «запись во внешнюю переменную», а
  собрать его вручную мешали статическое число потоков, один аргумент и
  отсутствие подсрезов. Обоснование стало правдой только с редукцией.)*
- **Асинхронность (async/await) — ОТЛОЖЕНА**, обоснование то же, что у каналов, и
  сильнее. Каналы отвергнуты за рантайм-библиотеку (мьютекс+condvar); async
  требует рантайма **несопоставимо большего** — цикл событий, epoll, таймеры,
  планировщик. Принять async, оставив каналы отложенными «из-за рантайма», было
  бы непоследовательно. Плюс async решает проблему, которой у языка ещё нет
  (I/O-конкурентность), при нерешённой той, что есть — пригодность имеющегося
  параллелизма. **Связка (не потерять):** если однажды рантайм для async всё же
  принимается, каналы автоматически теряют своё главное возражение и подлежат
  пересмотру — эти два решения стоят и падают вместе. Условие возврата: реальная
  I/O-задача **и** явное решение по рантайму. Роадмап — `ПЛАН_ПАРАЛЛЕЛИЗМА.md`.
- **`goto` — НЕ реализовывать** (сознательное решение). Конфликтует с ядром
  инвариантов Konda: definite-assignment (прыжок через инициализацию → чтение
  мусора), владение/деструкторы/autofree (прыжок из/в область минует
  RAII-деструкцию → утечка/double-free), move/borrow-анализ. Все текущие анализы
  **структурные** (по дереву if/while/for); безопасный `goto` потребовал бы
  CFG-анализа с потоками по произвольным рёбрам — крупная перестройка, высокий
  риск. Главный use-case («goto cleanup») в Konda **уже не нужен**: срезы
  autofree сами освобождаются при выходе из области. Безопасная альтернатива для
  выхода из вложенных циклов (помеченные `прервать`/`продолжить`) — возможна
  позже как отдельная структурная фича, но не `goto`.

## Приоритеты дальше (безопасность уже в хорошем состоянии — дальше полнота)

1. ✅ **Sum-типы с данными** — реализованы: `перечисление Имя { Вариант(тип…) … }`
   + привязка payload в ветках `выбор`; кодоген — размеченный союз (tag+union),
   нулевая рантайм-цена; поля варианта — примитивы/указатели (без `срез`).
   Дальше на их основе — `Результат<T,E>` и опция-над-значениями (`возможно<значение>`).
2. ✅ **Индексация произвольного lvalue** — реализована для срезов
   (`структура.поле[i]`, цепочки `а.б[i].в[j]`); проверка границ через `срез_доступ`.
   `функция()[i]` запрещён (утечка владеющего временного — присвоить в переменную).
3. ~~Транслитерация имён в ASCII~~ — **снято** (пользователь против манглинга/
   неявных имён; держим кириллицу как есть, C23 UTF-8-идентификаторы).
4. Полнота: обобщённые структуры, cross-file generics, массивы (вывод размера,
   локальный `срез<T> с = массив`), батч-ошибки парсера.

**Параллелизм — статус и роадмап: `ПЛАН_ПАРАЛЛЕЛИЗМА.md`** (там же — чего стоит
каждый следующий шаг и какой инвариант затрагивает). Кратко:
✅ **Потоки** (модель передачи владения, см. «Принятые решения»): `поток<T>`,
`запустить(ф, арг)`, `присоединить(п)` + **авто-join** дескриптора при выходе из
области (RAII, как autofree срезов).
✅ **`параллельно для`** — параллелизм данных: диапазон дробится на
непересекающиеся куски, общения между потоками нет вовсе. Гонки невозможны по
построению; проверено TSan (0 гонок на 10⁶ элементов).
✅ **Редукция** (`редукция(итог)`) — приватный аккумулятор на кусок + свёртка
после join; только целые (float не ассоциативен → результат зависел бы от числа
ядер). Ею map-reduce стал выразим.
✅ **Вызовы чистых функций** в теле `параллельно для` — чистота **выводится**
(глобалов нет → эффект возможен только через `изменяемый`/`вывод`/срез-параметр/
указатель/`небезопасно`/вызов нечистой), новых ключевых слов нет.
Каналы и async — **отложены осознанно** (см. «Принятые решения»).
⚠️ Ограничения v1 (не дыры в безопасности — всё ловится компилятором): число
потоков — константа в исходнике; поток, объявленный в цикле, последователен
(есть предупреждение); ≤ 1 аргумент; `срез` как результат не поддержан.

## Сборка, тесты, санитайзеры

```sh
make                       # сборка libkonda.so + CLI (должно быть 0 предупреждений)
cd tests && ./run.sh       # весь набор → «OK: все тесты прошли»
# ASan/UBSan-сборка CLI (все LIB_SRC + основа.c):
cc -std=gnu23 -g -O0 -fsanitize=address,undefined \
   дин_массив.c токенизатор_лексер.c конвейер.c ввод_вывод.c аст.c разбор.c \
   обобщения.c семантика.c владение.c кодоген.c заголовки.c интерфейс.c \
   libkonda_ide.c основа.c -o /tmp/kasan
```

Конвейер: `токенизатор_лексер.c` → `разбор.c` → `обобщения.c` (мономорфизация) →
`семантика.c` → `кодоген.c`; собран в `Собранное/libkonda.so`, CLI `основа.c`.
Детали — в `docs/АРХИТЕКТУРА_ТРАНСПИЛЯТОРА.md`, пользовательский обзор — в `README.md`.
