Компьютер — гениальный, но очень буквальный исполнитель

Представь робота-повара. Он приготовит что угодно — но только если объяснить ему предельно точно и на его языке. Скажешь «посоли по вкусу» — он растеряется. Скажешь «добавь 2 грамма соли» — сделает идеально.
Konda2

Программа — это рецепт для такого исполнителя 📋

Пошаговая инструкция, где каждый шаг понятен без догадок:
1. Спроси у человека имя  →  2. Если имя есть — поздоровайся  →  3. Иначе — скажи «привет, незнакомец»
Konda3

Проблема 😕

Свои «рецепты» компьютеру пишут на специальных языках программирования. Их сотни — но почти все они на английском:
if (count > 1) {
    printf("hello");
} else {
    printf("bye");
}
if, else, int, return, while… — чтобы программировать, сначала учишь чужие слова.
Konda4
А что если…

…писать программы на русском? 💡

тот же рецепт, но на Konda
если (количество > 1) {
    печать("привет");
} иначе {
    печать("пока");
}
Каждое ключевое слово — родное: если, иначе, целое, точка_входа.

Шаг 1: разбор на слова 🔤

Как на уроке русского разбирают предложение на части — Konda разбивает код на «кусочки» (их называют токены):
было
если (количество > 1)
стало
если(количество >1)

Суперспособность

Ловит ошибки ещё до запуска 🛡️

Как заботливый учитель, который проверяет работу до экзамена. Попросил у списка из одного элемента — второй? Konda сразу остановит и объяснит:
⚠️ Ошибка (строка 6): просишь элемент №1, а в списке точно есть только один. Так нельзя 🙅
Konda11

А зачем это всё? 🤔

  • Понять магию изнутри: увидеть, как текст становится программой.
  • Программировать на родном языке — без барьера чужих слов.
  • Это учебный проект: его делают, чтобы научиться, как устроены настоящие языки.

Что такое AST? 🌳

Абстрактное синтаксическое дерево — структура программы в виде дерева.
сейчас: плоская цепочка
2 + 3 * 4
➜
дерево: видна структура
+ / \ 2 × / \ 3 4
🌳 Дерево само «знает»: умножение ниже по ветке — значит, считается раньше. Это «схема предложения» из слайда про токены, только для кода.
